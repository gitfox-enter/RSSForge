### Bug Fix: RSS Timezone Issue
**Time**: 15:30-15:45  
**Priority**: P0  

**Symptoms**:
- User reported RSS feeds showing incorrect time
- Morning 8:00 displayed as afternoon 16:00
- Affecting all 48 feeds

**Investigation**:
```bash
# Checked raw feed
curl -s feeds/线报酷.xml | grep pubDate
# Output: Sat, 11 Jul 2026 14:38:42 -0000
```

**Root Cause**:
- `rss_feed.py` using `formatdate(localtime=False)`
- RFC 2822 `-0000` treated as UTC by readers
- +8 hour offset for Asia/Shanghai timezone

**Fix Applied**:
```python
# File: rss_feed.py
# Line: 47

# Before
pub_date = formatdate(localtime=False)

# After
from datetime import datetime
import pytz
tz = pytz.timezone('Asia/Shanghai')
pub_date = datetime.now(tz).strftime('%a, %d %b %Y %H:%M:%S +0800')
```

**Deployment**:
- Committed: f6d07d820d67be1c828559a1f9263b9467932010
- Pushed to: origin/main
- Verification: ✅ Feeds now show correct +0800 timezone

**Lessons Learned**:
- Always test RSS feeds with actual readers
- Timezone handling is tricky in Python
- Use explicit timezone strings, not offsets
