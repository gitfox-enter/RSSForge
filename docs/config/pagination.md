# 分页抓取与历史回填

首次添加一个站点时，网站通常已经有大量历史内容。RSSForge 支持分页抓取，把历史内容全部带回你的订阅源。

## 问题背景

大多数网站的首页只显示最新内容，更旧的内容在第 2、3、4... 页。如果只抓首页，历史内容就丢失了：

```
网站首页（抓取范围）：
  ✅ 第 1-30 条 — 今天和最近的内容
  ❌ 第 31-500 条 — 更旧的内容（被遗漏）
```

## 开启分页抓取

在 `sites.yaml` 中给站点加一行 `max_pages`：

```yaml
sites:
  - url: "https://www.example.com/"
    name: 示例论坛
    max_pages: 5    # ← 抓 5 页历史内容
    interval: 60
```

## 分页机制

当 `max_pages > 1` 时，爬虫会：

1. **抓取第 1 页** → 解析条目 + 检测「下一页」链接
2. **发现下一页** → 继续抓第 2 页
3. **重复直到达到 `max_pages` 或没有更多页面**

### 下一页检测逻辑

自动识别以下常见分页模式：

| 模式 | 示例 |
|------|------|
| 中文文字 | 「下一页」「下页」 |
| 英文文字 | `next` `older` `later` |
| 特殊符号 | `»` `›` `→` |
| HTML 属性 | `rel="next"` |
| CSS 类名 | `.pagination .next` `.pager li.next` |

## 示例：豆瓣小组历史回填

```yaml
sites:
  - url: "https://www.douban.com/group/711811/"
    name: 豆瓣某小组
    max_pages: 10    # 抓 10 页，大概 300 条历史帖子
    interval: 120
    tier: mid
```

## 自动去重

分页抓取时，同一 URL 的条目只保留一条，不会因为跨页而重复。

## 注意事项

- **首次回填会产生大量抓取**：分页越多，请求越多，首次运行时间会更长
- **建议设置 `interval` 较大**：首次回填完成后，日常用较长间隔即可
- **不是所有网站都有分页**：单页无限滚动（infinite scroll）的网站无法分页抓取
- **避免过大 `max_pages`**：设置 5-10 页通常足够，太大可能导致超时

## 建议配置

| 站点类型 | 建议 `max_pages` | 说明 |
|---------|-----------------|------|
| 论坛/社区 | 5-10 | 帖子价值高，值得保留更多历史 |
| 线报/新闻 | 1-3 | 更新频繁，历史价值不高 |
| 软件站 | 3-5 | 版本更新慢，历史版本有用 |
| 电商/价格监控 | 1-2 | 关注新品，历史价值有限 |
