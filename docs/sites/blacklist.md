# 黑名单站点

## 为什么站点会被列入黑名单？

站点进入黑名单的原因通常包括：

- **域名失效** — 网站已关闭、域名过期或被转卖
- **DNS 无法解析** — 域名不再指向任何服务器
- **服务器无响应** — 网站可以 ping 通但拒绝 HTTP 连接
- **持续返回错误** — 多次测试均返回 502/503/500 等服务器错误
- **反爬措施过于激进** — 持续返回 403/429 导致大量无效请求
- **被篡改或挂马** — 网站被黑、跳转至垃圾站

> 如果你发现某个已列入黑名单的站点已经恢复，可以在 [GitHub Issues](https://github.com/gitfox-enter/RSSForge/issues) 反馈，我们会重新测试并考虑恢复。

## 失效原因说明

### DNS / 连接失败

域名指向的服务器不再存在或网络不通。这是最常见的失效原因，通常意味着网站已经彻底关闭。

### 502 Bad Gateway

源站服务器在运行，但上游服务（如 CDN、负载均衡器）出现问题。可能是临时故障，也可能是网站已搬迁但旧域名未重定向。

### Connection Refused

服务器运行中，但 Web 服务（80/443 端口）拒绝连接。常见于：服务器宕机、Web 服务未启动、或防火墙主动拦截。

## 如何申请恢复？

如果你发现某个被禁站点已经恢复，可以：

1. 在 [GitHub Issues](https://github.com/gitfox-enter/RSSForge/issues) 新建 Issue，标题格式：`[站点恢复] 网站名 + 原URL`
2. 说明你手动访问该网站确认正常
3. 管理员测试后会从 `dead_sites` 中移除
