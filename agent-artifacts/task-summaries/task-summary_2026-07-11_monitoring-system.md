## 监控指标

### 实时指标（每天 4 次）
- ✅ Actions 运行成功率
- ✅ feeds_meta.json 更新延迟
- ⚠️ 健康站点占比（当前 25%）
- ❌ 失败任务错误类型

### 周期指标（每周）
- 总站点数变化趋势
- 新增/失效站点数量
- 本周新增 item 数量
- 高频/低频站点分布

## 异常处理流程

### 1. Actions 失败
1. 查看错误日志
2. 判断失败原因（网络/配置/代码）
3. 如果是代码问题，修复并提交
4. 如果是配置问题，更新 sites.yaml
5. 记录到 Issues

### 2. feeds_meta.json 超过 2 小时未更新
1. 检查 GitHub Actions 是否运行
2. 检查是否有并发冲突
3. 检查仓库状态
4. 必要时手动触发 workflow

### 3. 大量站点失效
1. 抽查 10 个站点 HTTP 状态
2. 区分域名失效/解析失败/正常
3. 移除确认失效的站点
4. 修复可修复的站点
