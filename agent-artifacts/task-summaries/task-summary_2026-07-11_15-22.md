# RSSForge 时间修复 + 全文功能方案

## 任务目标
修复 RSS feed 时间显示错误（多8小时），并添加全文抓取功能

## 已完成 ✅

### 1. 时间问题修复
**问题：** RSS feed 的 `pubDate` 使用 `-0000` 时区，RSS 阅读器误判为 UTC，导致时间多8小时
- 早上8点的文章，阅读器显示下午4点

**修复方案：**
- 修改 `rss_feed.py` 的 `_build_rss2_feed` 函数
- 将 `formatdate(utc_ts, localtime=False)` 改为 `dt.strftime('%a, %d %b %Y %H:%M:%S +0800')`
- 明确显示 `+0800` 时区

**提交记录：**
- Commit: [f6d07d8](https://github.com/gitfox-enter/RSSForge/commit/f6d07d820d67be1c828559a1f9263b9467932010)
- 文件: `rss_feed.py`
- 修改行数: 3行（删除 formatdate 逻辑，改用 strftime）

**验证：**
- 修改前: `Sat, 11 Jul 2026 14:38:42 -0000` → 被阅读器当作 UTC
- 修改后: `Sat, 11 Jul 2026 14:38:42 +0800` → 明确北京时间

## 待实现 ❌

### 2. 全文抓取功能
**需求：** RSS 阅读器里直接看到正文，不用跳转原文

**现状：**
- RSSForge 只抓列表页的标题和时间
- 不抓详情页正文
- 用户在 RSS App 里只能看标题，需要跳转才能看内容

**实现方案：**

#### 步骤 1: 修改 sites.yaml
给需要全文的站点添加配置开关：

```yaml
- url: "https://www.huifabu.cn/"
  max_pages: 3
  name: 汇发部
  tier: high
  interval: 20
  fast_check: true
  fetch_content: true  # 新增配置
```

#### 步骤 2: 修改 crawler/engine.py
在 `check_one_async()` 函数里添加正文抓取逻辑：

```python
# 位置：items 抓取成功后，返回结果前
if site_config.get('fetch_content'):
    for item in items[:10]:  # 限制前10条
        item_url = item.get('url', '')
        if item_url:
            try:
                success, result = await fetch_page_content_async(
                    item_url, session, old_records
                )
                if success and result.get('text'):
                    item['summary'] = result['text'][:500]  # 截取前500字
            except Exception as e:
                logger.warning(f"Content fetch failed: {item_url}: {e}")
```

**风险控制：**
- 只给少数站点开启（汇发部等）
- 正文抓取失败不影响标题和时间
- 限制抓取数量（前10条）
- 通过配置可随时关闭

**影响评估：**
- 正面：阅读体验更好，不用跳转
- 负面：
  - GitHub Actions 运行时间变长（每个条目多一次请求）
  - 部分站点可能触发反爬
  - 正文提取可能不准确
- 缓解：
  - 限制抓取数量
  - 只给可靠站点开启
  - 可随时关闭
